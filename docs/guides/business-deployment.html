2. Hardware Setup

Your AI agent needs a machine that stays on. Three options, from free to premium:

💻

Option A: Existing Laptop

$0

Use what you have. Windows (WSL2), Mac, or Linux. Set it to never sleep and plug it in. Good for testing and small teams.

🖥️

Option B: Beelink Mini PC

~$350–$400

Beelink SER5 MAX (AMD Ryzen 7, 16GB RAM) or EQ12 (Intel N100, 16GB). Silent, tiny, always-on. Perfect for a dedicated agent.

🍎

Option C: Mac Mini

$599

Apple M-series chip. Premium build quality, excellent power efficiency. Native macOS with great terminal support.

Minimum Requirements

  • CPU: 4 cores (any modern processor)
  • RAM: 8GB minimum, 16GB recommended
  • Storage: 128GB SSD (agents are lightweight)
  • Network: Stable internet connection (wired preferred)
  • Power: UPS recommended for business-critical deployments

Initial OS Setup

Windows (WSL2)

# Open PowerShell as Administrator
wsl --install -d Ubuntu
# Restart, then open Ubuntu from Start menu
# Disable sleep: Settings → System → Power → Never

Mac

# Disable sleep
sudo pmset -a disablesleep 1
# Enable auto-restart after power failure
sudo pmset -a autorestart 1

Linux

# Disable suspend
sudo systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target

3. Software Installation

Four commands. That's it.

Step 1: Install Node.js

curl -fsSL https://fnm.vercel.app/install | bash && fnm install 22

Or use the official installer at nodejs.org — download the LTS version.

Step 2: Install OpenClaw

npm install -g openclaw

Step 3: Install Origin Fortress

npm install -g origin-fortress

Step 4: Run the Origin Fortress Installer

curl -sSL https://origin-fortress.com/install.sh | bash

The installer will:

  • Configure security policies and permission tiers
  • Set up audit logging
  • Create your agent workspace at ~/.openclaw/workspace/
  • Generate a security report

Step 5: Set Your API Key

# Anthropic (recommended)
openclaw config set ANTHROPIC_API_KEY sk-ant-...

# Or OpenAI
openclaw config set OPENAI_API_KEY sk-...

Step 6: Verify Everything Works

openclaw status
origin-fortress scan

4. Agent Configuration

Your agent's behavior is defined by markdown files in ~/.openclaw/workspace/. No code required — just plain English instructions.

SOUL.md — Agent Personality

This is who your agent is. It defines tone, behavior, and boundaries.

~/.openclaw/workspace/SOUL.md — Finance Agent for Acme Corp
# SOUL.md — Acme Corp Finance Agent

You are Acme Corp's finance agent. You are precise, thorough,
and conservative with financial decisions.

## Core Behaviors
- **Always double-check numbers.** Never send a financial report
  without verifying totals.
- **Flag anomalies.** If an expense is 20%+ above average, alert
  the CFO immediately.
- **Be concise in reports.** Executives want the number, not the
  story behind it.
- **Never authorize payments** without explicit human approval.

## Tone
Professional but not stiff. You're a trusted colleague, not a
robot reading a script.

## Boundaries
- Never share financial data outside approved channels.
- Never modify accounting records directly.
- Always cite your sources (which spreadsheet, which email).

  Escalate to ops manager only if it requires budget approval

## Tone
Direct, action-oriented. Use bullet points. Skip pleasantries

5. Security Hardening

This is where Origin Fortress earns its keep. An unsecured AI agent with access to your business systems is a liability. Origin Fortress turns it into a trusted employee with clear boundaries.

Permission Tiers

Origin Fortress enforces three permission levels:

🔒

Worker

Read-only + safe tools. Can read files, search the web, check email. Cannot modify files, run shell commands, or access credentials. Best for: monitoring agents, research agents.

🛡️

Standard

Read-write with guardrails. Can edit workspace files, send messages, run approved commands. Blocked from system files, credentials, and destructive operations. Best for: most business agents.

⚡

Full

Unrestricted (admin only). Full system access. Use only for trusted admin agents with human-in-the-loop oversight. Requires explicit opt-in.

Forbidden Zones

Origin Fortress blocks agent access to sensitive paths by default:

# These paths are automatically protected:
~/.ssh/                    # SSH keys
~/.aws/                    # AWS credentials
~/.config/gcloud/          # Google Cloud credentials
~/.gnupg/                  # GPG keys
/etc/shadow                # System passwords
*.pem, *.key               # Private keys
.env                       # Environment secrets

# Add your own forbidden zones:
origin-fortress config add-forbidden "/path/to/financial-records"
origin-fortress config add-forbidden "/path/to/hr-data"

FinanceGuard

For companies handling financial data, Origin Fortress's FinanceGuard module adds extra protections:

  • PII detection: Automatically redacts Social Security numbers, credit card numbers, and bank account details from agent outputs
  • Transaction limits: Prevents the agent from initiating transactions above a configurable threshold
  • Segregation of duties: Ensures financial actions require human approval
  • SOX compliance logging: Every financial data access is logged with timestamps, user context, and justification
origin-fortress enable financeguard
origin-fortress financeguard set-limit --amount 1000 --currency USD
origin-fortress financeguard require-approval --actions "payment,transfer,refund"

McpFirewall

Control exactly which MCP tools your agent can access:

# Allow only specific tools
origin-fortress mcp-firewall allow "email:read,email:send"
origin-fortress mcp-firewall allow "calendar:read"
origin-fortress mcp-firewall deny "filesystem:delete"

# Review current firewall rules
origin-fortress mcp-firewall list

Audit Trails

Every agent action is logged for compliance review:

# View recent agent activity
origin-fortress audit log --last 24h

# Export audit log for compliance
origin-fortress audit export --format csv --from 2026-01-01 --to 2026-01-31

# Set up alerts for sensitive actions
origin-fortress audit alert --on "credential_access,file_delete,external_send"

6. Connecting Your Team

Telegram Bot Setup

  1. Open Telegram, search for @BotFather
  2. Send /newbot and follow the prompts to name your bot
  3. Copy the bot token
  4. Configure OpenClaw:
openclaw config set TELEGRAM_BOT_TOKEN "your-bot-token-here"
openclaw channel add telegram
openclaw gateway restart

Your team can now message the bot directly or add it to a group chat.

Slack Integration

  1. Go to api.slack.com/apps and create a new app
  2. Under OAuth & Permissions, add scopes: chat:write, channels:read, channels:history, users:read
  3. Install the app to your workspace and copy the Bot Token
  4. Configure OpenClaw:
openclaw config set SLACK_BOT_TOKEN "xoxb-your-token"
openclaw config set SLACK_APP_TOKEN "xapp-your-token"
openclaw channel add slack
openclaw gateway restart

Email Integration

# Configure IMAP for reading email
openclaw config set EMAIL_IMAP_HOST "imap.gmail.com"
openclaw config set EMAIL_IMAP_PORT 993
openclaw config set EMAIL_USER "agent@yourcompany.com"
openclaw config set EMAIL_PASSWORD "your-app-password"

# Configure SMTP for sending email
openclaw config set EMAIL_SMTP_HOST "smtp.gmail.com"
openclaw config set EMAIL_SMTP_PORT 587

openclaw channel add email
openclaw gateway restart
⚠️ Security tip: Use app-specific passwords (not your main password). For Gmail, enable 2FA and generate an app password at myaccount.google.com/apppasswords.

8. Maintenance & Monitoring

Keep Your Agent Running

# Start the agent as a system service (auto-restarts on crash/reboot)
openclaw gateway start

# Check status
openclaw gateway status

# View live logs
journalctl --user -u openclaw-gateway -f

Updates

# Update OpenClaw and Origin Fortress
npm update -g openclaw origin-fortress

# After updating, restart the gateway
openclaw gateway restart

# Re-run security scan after updates
origin-fortress scan

Review Audit Logs

# What did the agent do today?
origin-fortress audit log --last 24h --summary

# Did it access any sensitive data?
origin-fortress audit log --filter sensitive --last 7d

# Export for compliance review
origin-fortress audit export --format csv --to /tmp/audit-feb-2026.csv

Backup Agent Configuration

# Back up your entire agent workspace
tar czf agent-backup-$(date +%Y%m%d).tar.gz ~/.openclaw/workspace/

# Or use git (recommended)
cd ~/.openclaw/workspace && git init && git add -A && git commit -m "Agent config backup"

Monitoring Checklist

  • Daily: Check openclaw gateway status — ensure it's running
  • Weekly: Review origin-fortress audit log --last 7d --summary for anomalies
  • Monthly: Run origin-fortress scan and update software
  • Quarterly: Review and update SOUL.md, MEMORY.md, and permission tiers

9. Frequently Asked Questions

How much does it cost to run an AI agent?

Hardware is a one-time cost ($0–$599). The ongoing cost is your AI API usage — typically $50–$300/month depending on how active your agent is. Anthropic's Claude and OpenAI's GPT-4 are the most common choices. This is dramatically cheaper than hiring a human for the same tasks.

Is my company data safe?

Yes — and this is the key advantage of the OpenClaw + Origin Fortress approach. Your agent runs on your hardware, not in a shared cloud. Data never leaves your network unless you explicitly configure an external channel. Origin Fortress adds permission controls, forbidden zones, and audit trails on top.

Can the agent access our accounting software?

Yes, through MCP (Model Context Protocol) integrations. OpenClaw supports MCP servers for QuickBooks, Xero, spreadsheets, databases, and custom APIs. Origin Fortress's McpFirewall lets you control exactly which tools the agent can use.

What happens if the agent makes a mistake?

Origin Fortress's audit trail records every action, so you can always see what happened and why. For high-stakes operations (payments, external communications), configure the agent to require human approval before acting. Start with Standard permission tier and narrow as needed.

Can we run multiple agents?

Yes. You can run multiple OpenClaw instances on the same machine, each with its own workspace, personality (SOUL.md), and security policies. A finance agent and an operations agent can run side by side without interfering.

Do we need a dedicated IT person to manage this?

No. Once set up, the system is largely self-maintaining. Updates are a single command (npm update -g openclaw origin-fortress). Monitoring is a weekly 5-minute check. That said, we offer managed service plans if you prefer hands-off operation.

What if our internet goes down?

The agent will pause and resume when connectivity returns. OpenClaw handles reconnection automatically. For business-critical deployments, we recommend a backup internet connection (e.g., cellular failover).

Is this HIPAA/SOC 2/SOX compliant?

The on-premise architecture (your hardware, your data) is a strong foundation for compliance. Origin Fortress's audit trails, access controls, and FinanceGuard module support SOX and SOC 2 requirements. For HIPAA, additional configuration is needed — contact us for guidance.

How long does setup take?

A basic setup takes 1–2 hours. A fully configured business agent with custom workflows, security hardening, and team integrations typically takes 1–2 days. Our done-for-you service handles everything in 3–5 business days.
